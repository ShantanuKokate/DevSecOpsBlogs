Kubernetes Troubleshooting: Solving PVC Pending & Ingress Not Working

Kubernetes Troubleshooting PVC Ingress StorageClass

Recently, while practicing Kubernetes deployments, I ran into two real-world problems that tested my debugging skills. Let me walk you through how I solved them.

Problem 1: PVC (PersistentVolumeClaim) Stuck in Pending

When I deployed my application, everything seemed fine ‚Äî but the Pod stayed stuck in Pending state.

Running:

kubectl get pvc

I saw:

NAME        STATUS    VOLUME   CAPACITY   ACCESS MODES   STORAGECLASS   AGE
my-pvc      Pending

üí° What I figured out:

Kubernetes couldn't bind my PVC to a PV because I hadn't defined a StorageClass. Also, I mistakenly requested ReadWriteMany mode, which wasn't supported on my cluster.

‚úÖ Fix:

Installed a default StorageClass (in my case, I used hostPath for practice). Corrected the accessModes to ReadWriteOnce.

Updated PVC:

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi
  storageClassName: standard

After applying this, my Pod got scheduled successfully.

Problem 2: Ingress Not Routing Traffic

Next, I set up an Ingress for external access. But when I tried the URL in my browser, I got nothing.

Checked Ingress:

kubectl get ingress

It showed correctly, but still no response.

üí° What I figured out:

I hadn't installed an Ingress Controller. Kubernetes by default only provides the Ingress API object, not the controller. DNS wasn't resolving because I was testing locally without /etc/hosts configured.

‚úÖ Fix:

Installed NGINX Ingress Controller:

kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/cloud/deploy.yaml

Added an entry in /etc/hosts:

127.0.0.1 myapp.local

Updated my Ingress to use that host.

After this, my app was accessible via http://myapp.local. üéâ

Lessons Learned

  • PVC Pending usually means a mismatch between PVC request and available StorageClass.
  • Ingress won't work without a controller ‚Äì it's just an API object until you install one.
  • Always check logs with: 
    kubectl describe pod <pod-name>
kubectl describe ingress <ingress-name>

This was a great learning moment for me in Kubernetes. Debugging PVC and Ingress issues gave me more confidence in handling real cluster problems.
